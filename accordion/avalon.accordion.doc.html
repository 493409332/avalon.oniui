accordion使用说明

accordion组件

accordion组件配置项说明：
name value description
autoRun {Bool} 是否自动渲染
container || widgetElement {Element|JQuery|String} 组件容器
controlCls {String} 组件容器自定义className
currentTriggerCls {String} 组件激活时在标题区域标记的className，默认值为：“ui-accordion-active”
data {Array} 组件配置数据对象，格式为{title: string, content: {String|Object:Accordion}}
content {String|Object:Accordion}}
initIndex || active {Number|null} 初始展开第几个抽屉
mode {nav|caret|custom} 组件展开模式，取值说明："nav"=点击整个标题区域以展开面板，"caret"=点击标题区域中的图标以展开面板，"custom"=自定义面板展开方式
multiple || collapsible {Bool} 是否允许同时展开多个抽屉面板
template { header: {String}, panel: {String} } 配置抽屉标题、面板区域的html模板
trigger {String} 组件展开模式为自定义(mode="custom")时，配置指定触发面板展开的对象
triggerType || event {'click'|'mouseover'} 触发方式："click"=点击触发，"mouseover"=鼠标悬浮触发
width {Number|'100%'}v 组件宽度
beforeSwitch || onBeforeActive {Function} 组件面板展开前回调函数，若函数返回false则不展开面板，传入参数：index=面板序号，header=标题区域jQuery对象，panel=面板区域jQuery对象
onSwitch || onActive {Function} 组件面板展开后回调函数，传入参数定义同上
rendered //组件是否完成渲染 currentIndex //组件最新展开的抽屉序号 setData([{ title: '标题', content: '正文' }]); //重定义组件配置数据对象 refresh(); //手工刷新组件视图 accordion.refresh({ data: [{ title: '标题', content: '正文' }] }); //另一种手工刷新组件的方式，可以同时重定义组件配置数据对象 accordion.getCurrentHeader(); //获得当前展开的抽屉标题对象，仅在config.multiple == false时有效 accordion.getCurrentPanel(); //获得当前展开的抽屉面板对象，仅在config.multiple == false时有效 accordion.getHeader(1); //获得指定序号的抽屉标题对象 accordion.getPanel(1); //获得指定序号的抽屉面板对象 accordion.getLength(); //获得组件的抽屉数量 accordion.getStatus(1); //获得指定序号的抽屉面板展开(1)/收起(0)状态 accordion.switchTo(1); //切换抽屉展开
